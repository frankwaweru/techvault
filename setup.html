🏪 Store ⚙️ Admin 📖 Setup Guide
🔥 Firebase Integration

Connect Your Database

Follow these steps to connect TechVault to Firebase — free real-time database and image storage that works across all devices.

⏱️ ~10 min to complete
💳 Free Firebase tier
🌍 Real-time across devices
Firebase Project Setup
1
Create a Firebase Project
Go to the Firebase Console and create a new project for your TechVault store.
Firebase Console
Sign in with your Google account
Open Firebase Console →
1. Click Add project
2. Enter project name e.g. techvault-store
3. Disable Google Analytics (not needed)
4. Click Create project
2
Register a Web App & Get Your Config
Add a web app to get the Firebase config credentials you'll paste into your HTML files.
1. In your project dashboard, click the </> (Web) icon
2. Enter app nickname: TechVault Web
3. Leave "Firebase Hosting" unchecked
4. Click Register app
5. You'll see a config object — copy it 
// Your config will look like this — copy YOUR actual values:
const firebaseConfig = {
  apiKey: "AIzaSyXXXXXXXXXXXXXXXXXXXXXXXXXXXXX",
  authDomain: "your-project.firebaseapp.com",
  projectId: "your-project-id",
  storageBucket: "your-project.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:abcdef123456"
};
⚠️
Paste your real values into the firebaseConfig block at the top of both admin.html and index.html. Replace every placeholder value.
3
Enable Firestore Database
Firestore stores your product names, prices, descriptions and image URLs in real-time.
1. In the left sidebar click Build → Firestore Database
2. Click Create database
3. Choose Start in test mode (you'll secure it in Step 5)
4. Select a location closest to Kenya — choose europe-west1 or us-central1
5. Click Enable
4
Enable Firebase Storage
Storage holds your product images so they load fast from a CDN globally.
1. In the left sidebar click Build → Storage
2. Click Get started
3. Choose Start in test mode
4. Select the same location as Firestore
5. Click Done
5
Set Security Rules
Allow the store to read products publicly, while keeping write access open for now. You can add admin authentication later.
Firestore Rules 
// Firestore Database → Rules tab → paste this:
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /products/{document=**} {
      // Anyone can read (store visitors)
      allow read: if true;
      // Only allow writes from your IP or add auth later
      allow write: if true;
    }
  }
}
Storage Rules 
// Storage → Rules tab → paste this:
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /{allPaths=**} {
      allow read: if true;
      allow write: if true;
    }
  }
}
⚠️
For launch: these open rules are fine to get started. Before going fully public, add Firebase Authentication to your admin panel so only you can write. This guide will cover that in a future step.
6
Paste Your Config Into Both Files
Open admin.html and index.html in any text editor. Find the firebaseConfig block near the top and replace the placeholder values with your real credentials.
Look for this comment in both files:
// ✏️ PASTE YOUR FIREBASE CONFIG HERE

Replace the entire firebaseConfig = { ... } object with the one from Firebase Console.
Do this in both admin.html and index.html.
✅
Once your config is pasted, open admin.html in your browser, add a product, then open index.html — it should appear immediately on the store!
7
Deploy to GitHub Pages
Push all 3 files to your GitHub repo. Firebase works perfectly on GitHub Pages.
index.html — store
admin.html — admin panel
setup.html — this guide
GitHub Pages URL pattern:
🏪 Store: https://username.github.io/techvault/
⚙️ Admin: https://username.github.io/techvault/admin.html
💡
Add your GitHub Pages domain to Firebase's Authorized Domains: Firebase Console → Authentication → Settings → Authorized domains → Add domain.
🎉

You're All Set!

Your TechVault store is now connected to Firebase. Products you add in the admin panel appear on the store in real-time across all devices.
